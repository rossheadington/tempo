    Helper `_parse_set(token: str) -> StrengthSet | None`:
    - Weighted `WxR` (accept lowercase `x`, uppercase `X`, unicode `×`, whitespace around the separator): `_parse_set("55x8")` → `StrengthSet(weight_kg=55.0, reps=8, duration_s=None)`. `_parse_set("52.5x10")` → `weight_kg=52.5, reps=10`. `_parse_set("55 x 8")` → same as `55x8`.
    - Timed hold `M:SS` (single colon only): `_parse_set("1:00")` → `StrengthSet(weight_kg=None, reps=None, duration_s=60)`; `0:30` → `duration_s=30`; `2:15` → `duration_s=135`.
    - Bodyweight bare integer: `_parse_set("15")` → `StrengthSet(weight_kg=None, reps=15, duration_s=None)`.
    - Anything else (e.g. `"bogus"`, `"1:2:3"`, `"abc x 8"`) → `None`.

    Parser `parse_strength(path: Path) -> StrengthContext`:
    - Missing file → `StrengthContext(present=False, sessions=[], source_path=str(path))`. Never raises.
    - One session per `## ` header block. Header grammar: leading `YYYY-MM-DD`, optional ` HH:MM`, optional ` — Name` (also accept ` - Name` after the date+time; both em-dash `—` and ASCII ` - ` are valid separators between time and name). `name` is everything after the first separator, trimmed; if absent, `name=None`.
    - Header with unparseable date → entire session block (everything up to the next `## YYYY-MM-DD` header) is skipped. Other `# ` lines that are NOT session headers (e.g. `# Strength sessions`) are skipped.
    - Metadata lines between header and first ` - ` exercise bullet (any order): `rest: M:SS` → `rest_s` in seconds (e.g. `1:30` → 90); malformed → `None`. `notes: <text>` → `notes` verbatim (rest of line, trimmed). Any other `key: value` line → silently ignored.
    - Exercise bullet ` - <Name> [(Equipment)] [[GROUP]]: <set>[, <set>]...`:
      - Name = text up to first `(`, `[`, or `:` that introduces an annotation; trim trailing whitespace.
      - `(Equipment)` optional parenthesised; stored verbatim as `equipment`.
      - `[GROUP]` optional bracketed; stored verbatim as `superset_group`.
      - `: ` separator required to introduce sets.
      - Comma-split set tokens; each goes through `_parse_set`; `None` results are silently dropped; sets stored in source order on the exercise.
      - An exercise with zero parseable sets is still kept (`sets=()`).
    - Non-bullet, non-metadata, non-header lines between bullets are skipped (prose).
    - Sessions returned in source order.

    Rollup `strength_rollup(sessions: list[StrengthSession], today: date) -> StrengthRollup`:
    - Empty input → all counts/tonnage zero, `last_session_date=None`, `last_session_days_ago=None`, `last_session_name=None`.
    - Closed intervals (mirror heat): 7d = `[today-6 .. today]`, 14d = `[today-13 .. today]`, 28d = `[today-27 .. today]`.
    - Future-dated sessions defensively filtered from every window AND do not become `last_session_*`.
    - `*_count` = sessions whose date falls in the window.
    - `*_tonnage_kg` = sum of `weight_kg * reps` over every weighted set (`weight_kg is not None and reps is not None`) inside in-window sessions. Bodyweight + timed sets contribute 0 to tonnage but their parent sessions DO count toward `*_count`.
    - `last_session_date` = most recent in-past session date; `last_session_days_ago = (today - last_session_date).days`; `last_session_name` = that session's `name` (or `"unnamed"` if `None`).
