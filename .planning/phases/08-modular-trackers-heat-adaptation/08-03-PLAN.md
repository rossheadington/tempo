    RaceLink dataclass (frozen, slots): race: Race, activity_id: int | None, link_status: str. link_status is one of the four documented values; type is bare str per recovery.py:65 convention (NOT typing.Literal).

    link_races_to_activities(races: list[Race], conn: sqlite3.Connection) -> list[RaceLink]:
    - Returns a list parallel to the input races (same length, same order).
    - Performs exactly ONE DB query (or zero if the activity table doesn't exist): `SELECT day, activity_id FROM activity` — load all (day, activity_id) pairs into an in-memory dict[str, list[int]] (day → list of activity_ids on that day).
    - For each race:
      - If race.race_date is None → RaceLink(race, None, "unlinked_no_date")
      - Else: look up activities on race.race_date.isoformat() in the dict
        - 0 matches → RaceLink(race, None, "unlinked_no_match")
        - exactly 1 → RaceLink(race, activity_id, "linked")
        - 2+ matches → RaceLink(race, None, "unlinked_ambiguous")
    - No sport filter (CONTEXT.md explicit: a race could be ridden/swum/run).
    - Defensive sqlite_master check: if the activity table doesn't exist (fresh DB pre-Phase-3), skip the SELECT and treat the dict as empty — every race becomes unlinked_no_match (or unlinked_no_date if applicable). Never crash.
    - Returns an empty list when races is an empty list (no scan needed).
  </behavior>
