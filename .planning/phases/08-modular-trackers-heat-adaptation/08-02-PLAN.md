  <behavior>
    HeatSession dataclass (frozen, slots): date: date (REQUIRED — entries without parseable date dropped at parse time), type: str | None, duration_min: float | None, temp_c: float | None, hr_avg: float | None, notes: str | None.

    HeatContext dataclass (frozen, slots): present: bool, sessions: list[HeatSession] = field(default_factory=list), source_path: str | None = None. Mirrors RacesContext shape.

    HeatRollup dataclass (frozen, slots): today: date, last_7d_count: int, last_7d_minutes: float, last_14d_count: int, last_14d_minutes: float, last_28d_count: int, last_28d_minutes: float, last_session_date: date | None, last_session_days_ago: int | None.

    parse_heat behaviour:
    - Missing file → HeatContext(present=False, sessions=[], source_path=str(path))
    - File present, prose/headings/non-bullet lines → skipped
    - A bullet with at least one recognised key (date / type / duration_min / temp_c / hr_avg / notes) → candidate session
    - Bullet with leading date prefix (e.g. `- 2026-05-20 - type: sauna | duration_min: 20`) → uses the leading 2026-05-20 as session date when no `date:` key present
    - Bullet with neither a `date:` key nor a parseable leading-date prefix → returns None from _parse_heat_line (session dropped, NOT stored — this is the inverse of races behaviour, which keeps the race with race_date=None)
    - Unknown keys (like `hr_max:`) → silently ignored
    - Malformed lines → skipped, never raise

    heat_rollup behaviour: implements the closed-interval algorithm from RESEARCH.md section 1 lines 92-139 (cutoff_7 = today - timedelta(days=6) for a 7-date window; future-dated sessions filtered defensively; last_session_days_ago = (today - max_session_date).days, 0 if today). Empty sessions → all zeros + None for last_session_*. Sessions with duration_min=None count toward *_count but contribute 0 to *_minutes.
  </behavior>
