<objective>
Build the **wizard orchestration layer** on top of the Layer-0 primitives shipped by Plan 14-01. This plan adds:

- `runos/setup/prompts.py` ‚Äî thin typer-prompt wrappers (visible / hidden / yes-no / banner / instruction block) with consistent `[set]` / `[done]` / `[fresh]` indicators
- `runos/setup/wizard.py` ‚Äî top-level `run_wizard(settings, *, only, skip_garmin, skip_telegram, skip_scheduler, skip_bot_scheduler, skip_smoke, non_interactive) -> int` plus one function per step (welcome, db, content, strava, garmin, telegram, scheduler, bot-scheduler, smoke, finish) in the LOCKED order from CONTEXT ¬ß "Step list (LOCKED, in order)"
- Update `runos/setup/__init__.py` to re-export `run_wizard`
- A new `runos setup` command in `runos/cli.py` that delegates entirely to `run_wizard`
- `tests/test_setup_wizard.py` covering orchestration + flag behaviour + per-step delegation

Purpose: ship the user-facing surface ‚Äî one command (`runos setup`) ‚Äî that walks a new user from a fresh clone to a working `runos run-daily` (and optionally a running Telegram bot), with every credentialed step delegating to the existing CLI helpers (no duplicated auth handshake, no duplicated launchd plist render, no duplicated MFA prompt).

The orchestrator OWNS prompts, dispatch, the `[done]`/`[set]`/`[fresh]` decision tree, and the final smoke-test reporting. It owns NO `.env` I/O surface of its own (delegates to 14-01's `atomic_write_env`), and NO state-detection surface of its own (delegates to 14-01's `detect_install_state`).

Output:
- `runos/setup/__init__.py` (modified ‚Äî add `run_wizard` re-export)
- `runos/setup/prompts.py` (new)
- `runos/setup/wizard.py` (new)
- `runos/cli.py` (modified ‚Äî one new `@app.command("setup")`)
- `tests/test_setup_wizard.py` (new ‚Äî 12+ tests)
