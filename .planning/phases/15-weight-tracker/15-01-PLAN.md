    - `WeightEntry(date: date, weight: float, unit: str, notes: str | None, source_line: int)` — `unit` is lowercase `"kg"` or `"lb"`; `lbs` in source is normalised to `lb`; `notes` is verbatim free text after `| notes:` or `None`; `source_line` is 1-indexed.

    Helpers (module-level so they are unit-testable):
    - `_to_kg(weight: float, unit: str) -> float`: `unit == "kg"` → return `weight`; `unit == "lb"` → return `weight * 0.453592`; anything else → return `weight` unchanged (defensive — parser should never produce other units, but the rollup must not crash).
    - `_parse_entry_line(line: str, line_no: int) -> WeightEntry | None`: parses one ` - ` bullet. Returns None on malformed (the caller records `line_no` in `malformed_lines`).
      - Grammar: `- <YYYY-MM-DD>: <weight> [<unit>] [| notes: <free text>]`
      - Leading whitespace + literal `- ` are stripped before parsing. A line that doesn't start with `- ` after stripping leading whitespace → return None (NOT recorded as malformed — it's not an entry shape at all; only the caller decides).
      - Actually: per CONTEXT line 131-133, "Unknown bullet format (anything that doesn't match the entry grammar) → skipped." Treat any ` - ` bullet that doesn't match the date+float shape AS malformed (recorded). Treat non-bullet lines (no leading `- `) as silently ignored prose (NOT recorded).
      - Date: required; must `date.fromisoformat`; failure → return None (malformed).
      - Weight: required; must parse as `float`; failure → return None (malformed).
      - Out-of-range weight (kg-equivalent <20 or >500 — convert lb→kg before the range check): return None (malformed). This catches typos like `7.24 kg` and `724 kg`.
      - Unit: optional token after weight. Accepted: `kg`, `lb`, `lbs` (case-insensitive). `lbs` normalised to `lb`. Missing → default `kg`. Anything else → return None (malformed).
      - Notes: optional. Split on the FIRST `| notes:` substring (case-sensitive `notes:`); everything after is the notes verbatim, trimmed of trailing whitespace. Subsequent `|` pipes are part of the notes.

    Parser `parse_weight(path: Path) -> WeightContext`:
    - Missing file (`not path.exists()`) → `WeightContext(present=False, entries=(), path=None, malformed_lines=())`. Never raises.
    - Read the file as UTF-8; tolerate BOM and trailing whitespace.
    - Iterate lines with 1-indexed `line_no`.
    - Skip blank lines and header lines (lines starting with `#` after stripping leading whitespace). These do NOT go in `malformed_lines`.
    - Bullet lines (lines starting with `- ` after stripping leading whitespace): pass to `_parse_entry_line(line, line_no)`. None → append `line_no` to `malformed_lines`. Otherwise append the `WeightEntry` to a working list.
    - Non-bullet, non-blank, non-header lines: silently ignored (they're prose; don't record).
    - After all lines: dedup by date (latest-wins by file order — iterate forward, later same-date entries overwrite earlier in a dict keyed by `date`), then sort ascending by `date`, then convert to tuple.
    - Return `WeightContext(present=True, entries=tuple(deduped_sorted), path=path, malformed_lines=tuple(malformed_lines))`.

    Rollup `weight_rollup(entries: tuple[WeightEntry, ...] | list[WeightEntry], today: date) -> WeightRollup`:
    - Accept both tuple and list of entries (defensive — the context stores tuple but tests may pass lists).
    - Empty input → `WeightRollup(latest_entry=None, latest_kg=None, days_since_last=None, avg_7d=None, avg_28d=None, ewma_trend=None, delta_vs_28d=None, unit_mixed=False)`.
    - Filter to entries `<= today` (defensive against future-dated rows; mirrors strength/heat).
    - `latest_entry` = the entry with the maximum `date` ≤ today (which, after sort, is the last element of the filtered list).
    - `latest_kg` = `_to_kg(latest_entry.weight, latest_entry.unit)`.
    - `days_since_last` = `(today - latest_entry.date).days` (≥ 0).
    - `unit_mixed` = `len({e.unit for e in filtered_entries}) > 1` (True iff both kg and lb appear).
    - `avg_7d` = mean of `_to_kg(e.weight, e.unit)` over entries with `today - 7 < e.date <= today` (left-open right-closed). None if zero entries in window.
    - `avg_28d` = same with `today - 28 < e.date <= today`. None if zero entries in window.
    - `ewma_trend` = EWMA(alpha=0.1) iterated forward through ALL filtered entries in date order, seeded from the FIRST entry's `_to_kg(weight, unit)`. Recurrence: `trend_i = alpha * x_i + (1 - alpha) * trend_{i-1}`. Final value after the last entry is the result. None if zero entries.
    - `delta_vs_28d` = `latest_kg - avg_28d` if both are non-None, else None.

    Window semantics MUST be left-open-right-closed `(today - N, today]` so a same-day weigh-in always counts. Carefully: `today - N` itself is EXCLUDED. Use `e.date > today - timedelta(days=N) and e.date <= today`.
