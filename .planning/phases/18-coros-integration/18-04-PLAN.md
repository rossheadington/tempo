## Scope

Add the `## Coros (EvoLab)` section to the recovery report. Follows the existing 3-state degradation pattern (absent → omit, stale → nudge, current → render block).

## Files modified

- `runos/analysis/recovery.py`:
  - Import `from runos.analysis.coros_evolab import EvoLabDay, EvoLabContext, read_evolab`.
  - Import `from runos.units import format_pace` (for the threshold-pace + race-prediction rendering in user units).
  - `RecoveryAssessment` dataclass gains two fields: `evolab: EvoLabDay | None`, `evolab_present: bool`.
  - `assess_recovery_from_db(...)` accepts an optional `units: Units | None = None` parameter (or threads it through from preferences) — needed for pace formatting in the rendered block.
  - `_render_evolab_section(assessment, *, units, today) -> list[str]` — new function. 3-state rule:
    - `evolab_present is False` OR no rows → return `[]` (omits section).
    - `latest.day < today - timedelta(days=3)` → `["", "## Coros (EvoLab)", "", f"_Last EvoLab reading {N} days ago — wear the watch to refresh the dashboard._"]`.
    - Current → render the full block (see CONTEXT.md § Recovery-report integration for exact markdown).
  - `render_recovery_report(...)` calls `_render_evolab_section` after the Nutrition section in the section-order list.
- `runos/analysis/runner.py`:
  - `generate_recovery(...)` reads `evolab_ctx = read_evolab(conn)`; passes `evolab=evolab_ctx.latest, evolab_present=evolab_ctx.present` into `assess_recovery_from_db`; passes `units=prefs.units` for the renderer.
  - Same for `generate_all` if it threads recovery-report args.

## Implementation notes

- **Block format**: copy from CONTEXT.md § Recovery-report integration (REVISED for the real EvoLab payload — no race-predictions table, has lthr).
- **Threshold-pace rendering**: use `format_pace(latest.ltsp_s_per_km, units)`. Show it as `_cross-check vs preferences.md threshold_pace_`.
- **Threshold-HR rendering**: simple `f"{lthr} bpm"`. Show as `_cross-check vs preferences.md threshold_hr_`.
- **Stamina 7d delta**: compute by reading the most-recent value AND the value from ~7 days ago. If 7-day-ago value is missing, omit the delta. Read both via `read_evolab` (it returns all rows; lookup the day-7-ago row in Python).
- **VO2max**: render as `f"{vo2max:.1f} ml/kg/min"` if not None; omit the line if None.
- **Training load**: render as `f"{training_load}"` if not None; omit the line if None.
- **Per-line omission rule**: each line in the block omits if its value is None. Section header stays. If ALL values are None, fall through to absent (don't render section).
- **Tests** added to `tests/test_recovery.py`:
  - `test_recovery_omits_evolab_section_when_absent`
  - `test_recovery_emits_stale_evolab_nudge`
  - `test_recovery_renders_evolab_block_with_vo2max_stamina_load_lthr_ltsp`
  - `test_recovery_evolab_renders_pace_in_user_units` — write a fixture with `Units(distance="miles", pace="min_per_mile")`; assert the threshold pace appears as `M:SS /mi` not `/km`.
  - `test_recovery_evolab_omits_missing_lines` — fixture with only vo2max set; assert other lines absent but section header present.
