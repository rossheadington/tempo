## Implementation notes

- **Settings prefix**: `RUNOS_COROS_*` is the env-var pattern (the default `RUNOS_` prefix from `Settings.model_config` applies; no `validation_alias` needed).
- **SecretStr**: `coros_password` uses `SecretStr` so it's never printed accidentally. The connector accesses the raw value via `.get_secret_value()`.
- **CLI error mode**: match the Garmin commands EXACTLY for consistency. Users who've muscle-memoried `runos garmin <foo>` will type `runos coros <foo>` and expect the same UX.
- **README mention**: optional in this wave; if added, one line under "What it tracks" pointing at `docs/COROS.md`.
- **Backfill command**: deliberately NOT shipped in v1.7 per CONTEXT.md § Deferred. `runos coros sync` covers the recurring use case. Add `runos coros backfill --days N` in a later micro-phase if needed.
