## Implementation notes

- **Migration header**: include the same kind of explanatory comment block as `runos/migrations/0004_wellness.sql` (the "why" + the "what's rederivable" callout).
- **Migration transaction**: wrapped automatically by `db.migrate()`. No `BEGIN` / `COMMIT` in the file itself.
- **Payload shape (verified in 18-01)**: raw `evolab_dashboard` payload from `/analyse/query` has structure `data.t7dayList[]` — each entry has `happenDay` (YYYYMMDD int), `vo2max`, `staminaLevel`, `trainingLoad`, `lthr`, `ltsp`. The transform iterates `t7dayList[]` and upserts one `coros_evolab_day` row per `happenDay`.
- **happenDay parsing**: integer YYYYMMDD (e.g. `20260529`). Convert to ISO `YYYY-MM-DD` for the `day` PK.
- **ltsp unit conversion**: Coros likely reports threshold speed in m/s or km/h (verify against cygnusb/coros-mcp's parser by inspecting `coros_api.py` OR by examining a real captured payload). Normalise to seconds-per-km at parse time. Document the conversion factor in the transform module's docstring. If both forms appear plausible in the wild, default to m/s with a fallback heuristic (`if 1 < ltsp < 10: m/s ; if 5 < ltsp < 30: km/h ; else: malformed`).
- **NO race-prediction parsing**: those fields are not in the v1.7 endpoint surface. Don't try to parse them; don't add columns. If Coros adds them later we extend the schema.
- **NO recovery_pct parsing**: not in the endpoint. Same treatment.
- **stamina_level / training_load / lthr**: pass through as int. NULL if missing.
- **vo2max**: pass through as float. NULL if missing.
- **Defensive parsing**: any field missing from the payload → stays `None` in the row. Don't fabricate. Log a warning if the payload is structurally unexpected; skip; never raise.
- **Idempotence**: re-running the transform on the same raw produces the same row. `fetched_at` updates to the latest run.
- **Foreign key to date_spine**: ensure the `day` exists in `date_spine` before insert. If it doesn't (very fresh day, spine hasn't been extended yet), `runos/transforms/spine.py` will add it on its next run — order spine BEFORE evolab in `runs/transforms/runner.py`.
- **Analysis reader**: `read_evolab(conn)` returns ALL rows sorted ascending by day (small table; few hundred rows lifetime). The `latest` property is the most recent day with any non-NULL field.
