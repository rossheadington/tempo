## Implementation notes

- **Read cygnusb/coros-mcp** before writing code. Use WebFetch on https://raw.githubusercontent.com/cygnusb/coros-mcp/main/coros_api.py (or the equivalent module path in that repo — investigate the repo's tree) to confirm exact endpoint URLs, request body shapes, response shapes. CONTEXT.md gives the contract; cygnusb is the authoritative reference.
- **HTTP library**: prefer `requests` if already in deps (check `pyproject.toml`); else use `httpx`; absolute last resort `urllib`. No new heavy dependencies. Check `pyproject.toml` first and report which you used.
- **The MD5 hash** is `hashlib.md5(password.encode("utf-8")).hexdigest()`. Lowercase hex (Python's default). NOT a security claim — it's the protocol.
- **Login body shape** per cygnusb-reference: `{"account": "<email>", "accountType": 2, "pwd": "<md5-hex>"}`. Verify before locking in.
- **Token persistence** at `~/.runos/tokens/coros/token` (single file holding the bearer token as raw text — same shape as a Garmin token file). Atomic write: temp file in same dir, fsync, rename. Mode 0600. Directory mode 0700.
- **Sync lookback window** = 3 days (`SYNC_LOOKBACK_DAYS = 3`), matching the Garmin pattern.
- **Backfill default** = 60 days (`DEFAULT_BACKFILL_DAYS = 60`), matching the Garmin pattern.
- **One-shot refresh on 401**: a wrapper around each authenticated GET that, on a 401, calls `_login(...)` once, retries the original call, then propagates a `CorosAuthError` if the retry also fails. NEVER more than one re-auth attempt per call.
- **Raw payload upsert**: use the `RawWriter.put(endpoint, key, payload)` API. Key shape: ISO `YYYY-MM-DD` (the calendar date the payload is for). EvoLab dashboard pulls today's date; sleep/hrv/heart_rate walk the lookback window day-by-day.
- **Logging**: include the email's domain only (e.g. `user@***.com`) when surfacing auth errors. NEVER log the raw email, NEVER log the password, NEVER log the token.
