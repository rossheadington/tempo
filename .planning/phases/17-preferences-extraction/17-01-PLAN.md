## Implementation notes

- Mirror `tempo/analysis/races.py` for the lenient-parser shape (line-by-line, capture `malformed_lines`, never raise).
- Mirror `tempo/analysis/nutrition.py` for the `frozen=True, slots=True` dataclass shape and the `present: bool` flag.
- `_parse_threshold_pace` is the most non-trivial parsing logic ‚Äî write small focused unit tests for each accepted form (`4:00/mi`, `4:00/km`, `240 s/km`, `240`, malformed inputs).
- `Units` defaults to `("km", "min_per_km")` ‚Äî this preserves current behaviour for any user with no `## Units` section.
- Prose sections are everything that isn't `Physiology` / `Units` / `Nutrition` (case-insensitive header match). They go into `prose_sections: dict[str, str]` keyed by `header.lower().strip()`.
- The parser MUST tolerate the H1 `# Preferences` line and any preamble before the first H2 ‚Äî those go into neither typed nor prose buckets, they're discarded.
- Inline comments: a `#` outside of a markdown header context strips the rest of the line. Be careful: `## Physiology` is a header, not a comment. The rule: only strip `# ` (with space) or end-of-line `#...` inside a typed-section line.
