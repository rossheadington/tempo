## Interface notes

- `Units` type is imported from `runos.analysis.preferences` (the dataclass produced by 17-01). 17-02 may need to declare a `TYPE_CHECKING` import or use a string forward-ref — that's fine. The two modules are designed to compose: `format_distance(act.distance_m, prefs.units)`.
- `format_distance(None, ...)` → `"–"` (en-dash, U+2013).
- `format_distance(-5, ...)` → `"–"` (negative values invalid).
- `format_distance(metres, units, precision=2)` → e.g. `"12.88 km"` or `"8.00 mi"`.
- `format_pace(s_per_km, units)`:
  - `None`, `<=0`, NaN → `"–"`.
  - Convert to total seconds in the target unit, then format as `M:SS /km` or `M:SS /mi` (zero-padded seconds).
  - Edge case: `s_per_km = 270` → `4:30 /km` and `s_per_km = 270` → `7:14 /mi` (because 270 s/km × 1.609344 km/mi = 434.5 s/mi = 7:14.5 → round to 7:15? or 7:14?). DECISION: round to nearest second. Document in module docstring.
- All conversions take only `s/km` as the canonical input — the renderer doesn't take "miles" as input, ever. Storage is SI; rendering is the preference.

## Test surface (mandatory)

Per CONTEXT.md § Test scope under `tests/test_units.py`. ~9 tests. Cover:
- Distance round-trip (`miles_to_km(km_to_miles(x)) == x` modulo float epsilon).
- Pace pivot point (`format_pace(240, Units(pace="min_per_km"))` == `"4:00 /km"` AND `format_pace(240, Units(pace="min_per_mile"))` == `"6:26 /mi"` since 240 s/km × 1.609344 = 386.24 s/mi = 6:26.24 → 6:26).
- Seconds rollover (`format_pace` handles values producing `MM:SS` where SS = 60 cleanly).
- None / zero / negative inputs.
