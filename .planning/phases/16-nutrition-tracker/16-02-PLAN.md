    `render_nutrition(today: date, rollup: NutritionRollup, today_breakdown: DailyNutrition, blocks_today: tuple[MealBlock, ...], context: FoodContext) -> str`:

    Builds the report markdown body as a list of lines, joined with `\n`. Five `## ` sections in order: `## Today's totals`, `## Per-meal breakdown`, `## 7-day rolling average`, `## 28-day kcal mean`, `## Goal`.

    **Header (above first `## ` section):**
    - Title: `# Nutrition — {today.isoformat()}` (mirrors the load-trend / recovery title style).
    - Freshness banner: build via the `freshness_header` helper from `runos/analysis/report.py` if its signature accepts a per-source line; otherwise inline a `Data: ...` line:
      - If `context.present` is True AND `context.entries` is non-empty: `Data: food.md present ({entries_count} entries across {distinct_days} days, last {latest_date.isoformat()})`. `distinct_days` = number of unique dates in `context.entries`; `latest_date` = `max(e.date for e in context.entries)`.
      - If `context.present` is True AND `context.entries` is empty: `Data: food.md present (0 entries — file exists but empty or all malformed)`.
      - If `context.present` is False: `Data: food.md absent` — return EARLY here with a single-paragraph body: `_No food log found. Create <content_root>/food.md to enable nutrition tracking. See docs/NUTRITION.md for the format._` (no `##` sections at all). The path string "<content_root>" is literal — do NOT interpolate; the docs handle the redirect explanation.

    **`## Today's totals`:**
    - `today_breakdown.entry_count == 0` → emit ONLY `_No entries logged for today yet._`.
    - `today_breakdown.entry_count > 0` → emit a single line:
      ```
      P:{protein_g:.0f}g · C:{carbs_g:.0f}g · F:{fat_g:.0f}g · cal:{kcal} ({pct_p:.0f}/{pct_c:.0f}/{pct_f:.0f} P/C/F %)
      ```
      Where the macro grams round to whole numbers (use `:.0f`) and percentages round to whole numbers. Use the literal `·` (U+00B7) middle-dot separator to mirror the recovery weight-section style.

    **`## Per-meal breakdown`:**
    - `today_breakdown.entry_count == 0` → OMIT THE SECTION ENTIRELY (no `## ` header, no body).
    - Otherwise → group `entries` (from the context, filtered to `entry.date == today`) by `meal_name`. For each unique `meal_name` in file order (use the first `source_line` per meal_name to order), emit:
      ```
      ### {meal_name or "(unspecified)"}
      - {food_label} — P:{protein_g}g · C:{carbs_g}g · F:{fat_g}g · cal:{kcal}
      - ...
      ```
      Format the macros at one-decimal precision (`:.1f` for P/C/F, integer for kcal) to match the source-file precision (e.g. `1.3g` banana protein).

    **`## 7-day rolling average`:**
    - `rollup.avg_7d is None` (i.e. `days_logged_7d == 0`) → emit `_No entries in the last 7 days._`
    - Otherwise → emit:
      ```
      P:{avg_p:.0f}g · C:{avg_c:.0f}g · F:{avg_f:.0f}g · cal:{avg_kcal} ({days_logged_7d} days logged of 7)
      ```
      Then a second line with the macro percentages:
      ```
      Macro split: {pct_p:.0f}% P · {pct_c:.0f}% C · {pct_f:.0f}% F
      ```

    **`## 28-day kcal mean`:**
    - `rollup.avg_28d_kcal is None` → emit `_Insufficient history._`
    - Otherwise → emit `{avg_28d_kcal} kcal/day`.

    **`## Goal`:**
    - `rollup.target_kcal is None` → OMIT THE SECTION ENTIRELY.
    - Otherwise → emit:
      ```
      Target {target_kcal} kcal/day · 7d delta {sign}{abs_delta} kcal/day
      ```
      Where `sign` is `+` for surplus, `−` (Unicode U+2212) for deficit, and `±` (U+00B1) for exact zero. `abs_delta = abs(rollup.deficit_surplus_7d)`. If `rollup.deficit_surplus_7d is None` (degenerate — should not happen when `target_kcal` is set AND `avg_7d` is set, but defensive): emit `Target {target_kcal} kcal/day · 7d delta n/a (insufficient 7d history)`.

    Trailing blank line at end-of-document (mirrors the other reports).
