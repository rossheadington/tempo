<objective>
Wire the nutrition parser+rollup from Plan 16-01 into the recovery report. Extend `RecoveryAssessment` with `nutrition` / `nutrition_present` fields, add a `_render_nutrition_section` helper that follows the same 3-state degradation rule as heat/strength/weight (absent → omit / stale → one-line nudge / current → full 7-day-trailing rollup line), thread `food_path` + `target_kcal` through `assess_recovery_from_db`, `runner.generate_recovery`, `runner.generate_all`, and the `tempo analyze recovery` CLI call site.

The staleness threshold is `> 3 days` (not 14 like weight) — food is logged daily, a 3-day gap makes the 7-day rollup unreliable (CONTEXT line 186, 296).
