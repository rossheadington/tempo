    - Missing file (`not path.exists()`) → `FoodContext(present=False, entries=(), blocks=(), path=None, malformed_lines=())`. Never raises.
    - Read file as UTF-8; tolerate BOM, trailing whitespace, mixed line endings.
    - Iterate lines with 1-indexed `line_no`.     - Line dispatch:
      - Blank line → ignore. Does NOT close the active block (blocks span blank-line separators in CONTEXT line 70 example — a blank line between block bullets is permitted).
      - Line starting with `## ` (after stripping leading whitespace) → CLOSE any active block (finalise to `MealBlock` if it had entries), then attempt `_parse_block_header`. On success → open a new active block. On failure → record header line as malformed AND mark a "dead block" so subsequent nested bullets through the next `##` header are captured as malformed.
      - Line starting with `# ` (single hash, NOT `##`) → ignore (comment).
      - Line starting with `- ` (after stripping leading whitespace):
        - If an active VALID block is open → call `_parse_block_bullet(line, line_no, block_date, block_meal)`. None → record `line_no` as malformed. Otherwise → append to the active block's entries.
        - If an active DEAD block is open → record `line_no` as malformed (the whole block is poisoned).
        - If no active block is open → call `_parse_inline_entry(line, line_no)`. None → record `line_no` as malformed. Otherwise → add to a working list of inline entries.
      - Any other non-blank, non-comment line → silently ignore (prose).
